# Аутентификация и первый запрос к API

В данном руководстве описан процесс получения доступа к API Сервиса и отправки первого тестового запроса для проверки статус-кодов.

## Шаг 1. Получение API-ключа

Для взаимодействия с API вам потребуется уникальный токен доступа (API Key).
1. Перейдите в личный кабинет разработчика.
2. В разделе **Настройки доступа** нажмите кнопку **Создать токен**.
3. Скопируйте сгенерированную строку. 

> [!WARNING]
> Никогда не передавайте токен третьим лицам и не коммитьте его в открытые репозитории.

## Шаг 2. Формат запроса

Все запросы к API должны отправляться по протоколу HTTPS методом `POST` на базовый URL: `localhosthehehe.com/ustalpisat/proshuvozmitenarabotu/yabuduochenhoroshimrabotnikom`.

В заголовках (Headers) запроса необходимо передать:
* `Content-Type: application/json`
* `Authorization: Bearer <ваш_токен>`

### Пример тела запроса (JSON)

```json
{
  "request_id": "req_987654321",
  "payload": {
    "target_id": 1024,
    "action": "get_status"
  }
}

**## Шаг 3. Обработка ответа**

В случае успешного выполнения запроса сервер возвращает статус-код 200 OK и JSON-объект с результатом операции.
Пример успешного ответа
{
  "status": "success",
  "timestamp": "2026-06-25T11:45:00Z",
  "data": {
    "is_active": true,
    "last_update": "2026-06-25"
  }
}
Возможные ошибки

Если запрос не выполнен, обратите внимание на код ответа:

401 Unauthorized — неверный или просроченный токен в заголовке Authorization.
400 Bad Request — передан некорректный JSON или отсутствуют обязательные поля в payload.
